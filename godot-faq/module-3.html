子弹设计与发射逻辑

Area2D 感应体设计、方向计算与父子运动解耦

📋 特性分析 Features
  • 穿透感应：子弹不应像实体墙一样撞停敌人，而是"重叠"后触发事件。
  • 独立运动：子弹发射后，其运动轨迹应独立于玩家（玩家移动，子弹不随动）。
  • 寿命管理：飞出范围或存活时间过长需销毁。
🌲 节点树构成 Node Tree

在独立场景 Bullet.tscn 中构建：

Node2D （根节点：负责基础位移逻辑） ├── Sprite2D （视觉图片） ├── Area2D （感应核心，负责重叠检测） └── CollisionShape2D （感应范围）

⚙️ 代码处理 Logic

在 Player.gd 发射子弹：

# Player.gd
func shoot():
    var target = get_nearest_enemy()
    if target == null: return

    var bullet = bullet_scene.instantiate()
    bullet.global_position = $Muzzle.global_position

    # 向量计算：计算指向目标的归一化方向
    bullet.direction = (target.global_position - global_position).normalized()

    # 注意：加到 parent (World) 而非 self，实现运动解耦
    get_parent().add_child(bullet)
📌 老师注记
  • Area2D vs CharacterBody2D：子弹用 Area2D 是因为只需要"感知重叠"，不需要物理碰撞反馈
  • normalized()：将方向向量转为单位向量（长度=1），保证子弹速度在各方向一致
  • add_child 到 parent：如果加到 Player 节点下，子弹会跟着 Player 一起移动
