# L6 教学 Brief：升级选择界面

> 这是**给 OpenClaw 生成教学材料的 brief**，不是给学员的 cheatsheet。
> 不写代码 / 不写完整 API 解释，只写**教学框架 + 认知路径**。
> 带课老师：朱古力（已回归本课堂）。

---

## 目标

做一个击杀数达到目标后的升级选择：
**击杀数够了 → 游戏 freeze → CanvasLayer 弹出左右两个升级选项 → 玩家选择一个 → Player 获得对应能力 → 界面消失 → 游戏继续。**

---

## 已有基础（学员已经会的，不要重新讲）

- 击杀数记录
- HUD 显示
- CanvasLayer / Control / Button / Label
- show / hide
- Button pressed
- @export 拖节点
- 修改 Player 变量

**所以本节课不要重新讲 HUD 和击杀数。**

---

## 本节课的最小增量

**只新增两个能力**：

1. 让游戏世界**整体 freeze**
2. 让升级选择 UI 在 freeze 时**仍然可以点击**

**Godot 对应机制**：

- `get_tree().paused = true / false`
- UpgradePanel → Process Mode → Always

**核心理解**（一句话）：

> **世界暂停，菜单继续。**

---

## 建构主义展开路径

**不要直接讲"今天学 pause 和 process_mode"**。

**从目标出发**：

> 我们想做"升级"。

**先让学员判断已有知识能做到哪些部分**：

- 什么时候升级：击杀数 + 判断
- 升级界面：CanvasLayer + Panel / Button
- 知道选了哪个：左右按钮各自的 pressed
- 加能力：修改 Player 的变量
- 显示 / 隐藏：show / hide

**然后暴露两个卡点**：

### 卡点 1：升级界面出现时，游戏世界还在动

如果让 Player、Enemy、Bullet、Spawner 各自停下来，会很麻烦、容易漏。
所以需要一个"**整体 freeze**"的机制。

### 卡点 2：世界 freeze 后，升级按钮还要能点

如果整个游戏都停了，按钮也停了，玩家就没法选择。
所以需要一种"**暂停时仍然工作的 UI**"。

```
Main
├── World
│   ├── Player
│   ├── Enemies
│   └── Bullets
├── HUDLayer
│   └── HUD
└── UpgradeLayer
    └── UpgradePanel
        ├── DimBackground
        ├── LeftBox
        │   └── LeftButton
        └── RightBox
            └── RightButton
```

---

## UI 设计要求

- UpgradePanel **平时隐藏**
- 击杀数达到目标时：
  - 游戏 freeze
  - UpgradePanel 显示
  - 背景变暗（DimBackground）
  - 左右两个选项出现
- 玩家点击左边或右边后：
  - 代码知道选了哪个
  - 修改 Player 的对应能力
  - UpgradePanel 隐藏
  - 游戏恢复

## 教学重点

**这节课不是"教 API"。**

**真正要学生理解的是**：

- 有些节点属于**游戏世界**，暂停时应该停
- 有些节点属于**菜单界面**，暂停时还要工作
- 升级选择界面就是"**暂停时还要工作的菜单节点**"

## 这个抽象的可迁移性

**这个结构以后可以迁移到**：

| 场景 | "世界"的状态 | "菜单"做的事 |
|------|------------|------------|
| **暂停菜单** | paused | "继续 / 退出"两个按钮 |
| **装备选择** | paused（选完才能动） | 左右两个装备卡槽选择 |
| **天赋选择** | paused | 升级 / 重置 / 切换 |
| **背包界面** | paused（查看 / 切换装备） | 物品 grid + 拖拽 |
| **结算界面** | paused（看本局结果） | "再来一局 / 返回主菜单" |

**L6 教的不是"升级"这一个功能**，是**"如何让一个暂停中仍能交互的菜单出现"**这个**通用模式**。
