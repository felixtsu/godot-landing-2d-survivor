导弹发射与命中爆炸

分三阶段掌握导弹飞行、爆炸效果、碰撞检测的完整流程

📋 特性分析 Features
  • Z键触发：按下 Z 键实例化一枚 Missile，方向朝向最近敌人。
  • 导弹飞行：Missile 以 direction × speed 每帧飞行，方向在发射时锁定。
  • 两种爆炸触发：1秒超时自动爆炸，或飞行中碰到敌人立即爆炸。
  • 爆炸效果：explode() 在当前位置生成 Explosion 场景，0.2秒后自动删除。

🌲 节点树构成 Node Tree

在 Player 场景中添加（作为子节点）：

Player（CharacterBody2D） （玩家角色） ├── Sprite2D （外观） ├── CollisionShape2D （碰撞体） ├── Timer （导弹发射冷却，可选） └── ...（其他节点）

Missile 和 Explosion 是独立场景，不放在 Player 树下，通过代码动态实例化。

Missile（Area2D） （导弹独立场景） ├── Sprite2D （导弹外观） ├── CollisionShape2D （碰撞检测） └── Timer （1秒后自动爆炸）
Explosion（Node2D） （爆炸效果独立场景） ├── Sprite2D / AnimatedSprite2D （爆炸动画） └── Timer （0.2秒后删除自己）

下面是浓缩版 6 步快速参考，详细步骤见上方子模块页。

第一步：创建 Missile 场景

1. 项目 → 场景 → 新建场景
2. 根节点选 Area2D，命名为 Missile
3. 添加子节点：Sprite2D + CollisionShape2D + Timer
4. 给 Timer 设置 Wait Time = 1.0，One Shot = On
5. 保存为 missile.tscn

第二步：创建 Explosion 场景

1. 项目 → 场景 → 新建场景
2. 根节点选 Node2D，命名为 Explosion
3. 添加子节点：AnimatedSprite2D（或 Sprite2D）+ Timer
4. 给 Timer 设置 Wait Time = 0.2，One Shot = On
5. 保存为 explosion.tscn

第三步：编写 Missile.gd

extends Area2D

var speed: float = 400.0
var direction: Vector2 = Vector2.ZERO
var explosion_scene: PackedScene = preload("res://explosion.tscn")

func _physics_process(delta):
	position += direction * speed * delta

func _on_timer_timeout():
	explode()

func _on_body_entered(body):
	explode()

func explode():
	# 生成爆炸效果
	var explosion = explosion_scene.instantiate()
	explosion.global_position = global_position
	get_tree().root.add_child(explosion)
	# 导弹消失
	queue_free()

第四步：在 Player.gd 添加发射逻辑（Z键）

extends CharacterBody2D

var missile_scene: PackedScene = preload("res://missile.tscn")

func _input(event):
	if event.is_action_pressed("ui_accept"):  # Z键（Godot默认ui_accept映射到Z）
		fire_missile()

func fire_missile():
	# 找出最近的敌人
	var enemies = get_tree().get_nodes_in_group("enemies")
	if enemies.is_empty():
		return
	var closest = enemies.reduce(func(a, b):
		return a if a.global_position.distance_to(global_position) < b.global_position.distance_to(global_position) else b
	)
	var dir = (closest.global_position - global_position).normalized()
	# 实例化导弹
	var missile = missile_scene.instantiate()
	missile.direction = dir
	missile.global_position = global_position
	get_tree().root.add_child(missile)

第五步：给敌人打上 Group 标签

1. 在 Enemy 场景的根节点上右键
2. 选择"为节点设置组..."
3. 输入 "enemies" → 确定

第六步：F5 测试

1. F5 运行游戏
2. 按 Z 键发射导弹
3. 导弹飞向最近敌人，命中后爆炸 = 成功！
🏗️ 架构：独立场景 × 动态实例化
  • Missile / Explosion 独立场景：不嵌入 Player 树，通过 preload + instantiate() 动态创建。
  • 方向锁定：direction 在 fire_missile() 时计算并传给 Missile，发射后不再改变。
  • 两种爆炸触发：Timer.timeout（时间到达）和 body_entered（碰撞到达），满足任一都会执行 explode()。
  • queue_free() = 消失：从场景树移除对象，不再占用性能。

口诀：Z键发射 → 锁定方向 → 飞向敌人 → 爆炸消失

📌 老师注记
  • 方向用 normalized()：missile.direction 必须 normalize，否则速度会因距离变化而不一致。
  • add_child 到 root：missile 和 explosion 要加到 get_tree().root（主场景根节点），不要加到 Player 下，避免随 Player 移动。
  • Timer 的 One Shot：爆炸 Timer 设为 One Shot = On，爆炸一次就够了，不要重复触发。
  • Missile 的 CollisionLayer/Mask：确保 Missile 的 Mask 包含 "enemies" 层，否则 body_entered 收不到信号。
  • reduce 找最近敌人：如果学生不理解 reduce，可以先用循环比较距离，原理一样。
⚠️ 常见坑点 Common Pitfalls
  • 方向没 normalize：direction 直接用 (closest - missile).normalized() 而不是 (closest - missile)，速度会忽快忽慢。
  • Missile 随 Player 旋转：Missile 的父节点如果是 Player 且 Player 有旋转，方向会叠加。需要把 Missile add_child 到 root 而不是 Player 下。
  • 爆炸位置偏移：用 global_position 而不是 position，确保爆炸效果出现在世界坐标的正确位置。
  • 碰撞检测没响应：检查 Missile 的 CollisionShape2D 是否已添加，以及 Mask 是否包含 enemy 层。
  • 导弹穿墙：如果 Missile 用 Area2D 而非 CharacterBody2D，它不会自动做物理碰撞，会穿过 TileMap。需要用 CharacterBody2D + move_and_collide() 才能被墙挡住。
💡 核心概念
  • 独立场景 + instantiate() = 可复用的对象池：Missile 和 Explosion 不写在 Player 里，改为独立场景按需创建
  • direction × speed × delta = 飞行：每帧移动一定距离，方向锁定后不改变
  • Timer + body_entered = 两种爆炸条件：时间到达或碰撞到达，满足任一就爆炸
  • queue_free() = 性能清理：爆炸后删除自己，避免导弹和爆炸残留积累
