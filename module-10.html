模块十

世界暂停，菜单继续

升级选择、暂停菜单、装备界面、结算画面 —— 共享同一个"暂停 + 菜单"模式

📋 核心概念 Concept
  • 两套世界：游戏世界（玩家、敌人、子弹、计时器）暂停时要停；菜单世界（升级选择、暂停面板、装备界面）暂停时还要工作。
  • 全局开关：get_tree().paused = true / false 一行代码控制整个游戏树暂停 / 恢复。
  • 白名单豁免：节点上的 Process Mode = Always 让该节点在暂停时仍然工作（UI 按钮、动画、计时都要这个）。
  • 分离的代价：用 CanvasLayer（module-8 已学）做菜单根节点，让 UI 永远在屏幕最上层，跟游戏世界不混在一起。

一句话：世界暂停，菜单继续。

🌲 节点树构成 Node Tree

在 Main 场景中添加：

Main（Node2D） ├── World（Node2D） （游戏世界：暂停时全停） │ ├── Player │ ├── Enemies │ └── Bullets ├── HUDLayer（CanvasLayer） （module-8 已学，HUD 显示） │ └── HUD └── UpgradeLayer（CanvasLayer） （菜单世界：暂停时仍工作） └── UpgradePanel （⚙️ Process Mode = Always） ├── DimBackground（ColorRect） （半透明遮罩） ├── LeftBox（PanelContainer） │ └── LeftButton（Button） └── RightBox（PanelContainer） └── RightButton（Button）

⚠️ 关键：UpgradePanel 节点要设 Process Mode = Always（在 Inspector 顶部，**不是** CanvasLayer 上设）。

参考结构，不是唯一答案。Container 类型（PanelContainer / VBoxContainer / HBoxContainer）学员可自由组合。

⚙️ API 速查 Quick Reference

暂停 / 恢复整个游戏

# 暂停
get_tree().paused = true

# 恢复
get_tree().paused = false

# 切换（任意脚本里都能调）
get_tree().paused = not get_tree().paused

⚠️ get_tree() 任何 Node 都有，返回的是 SceneTree（整棵游戏树）。

Process Mode 5 个值（节点属性，Inspector 顶部下拉）

INHERIT    （0）  默认：跟随父节点的设置
PAUSABLE   （1）  跟随全局 paused（普通节点用这个）
WHEN_PAUSED（2）  只在 paused 时跑（暂停菜单用这个）
ALWAYS     （3）  不受 paused 影响（升级弹窗用这个）
DISABLED   （4）  永远不跑（临时禁用）

⚠️ 在 Inspector 顶部下拉框选；脚本里用 Node.PROCESS_MODE_ALWAYS 等常量。

运行时改 Process Mode

# 任何节点都可以在 _ready() 里设置
func _ready():
    process_mode = Node.PROCESS_MODE_ALWAYS

「世界暂停 + 菜单继续」是模块 10 的核心模式，**不是**"只做升级选择"。所有"玩家要看 / 选 / 操作但游戏要停"的界面都用这套：

  • 暂停菜单：玩家按 ESC → 暂停 → 显示"继续 / 退出"按钮 → 玩家选择 → 恢复或退出
  • 装备选择：在指定节点上拾取装备 → 暂停 → 弹出"换下/换上"对比 → 选择 → 恢复
  • 天赋加点：升级或关卡结束 → 暂停 → 显示天赋树 → 加点 → 恢复
  • 背包 / 商店：打开背包 → 暂停 → 拖拽物品 / 买 / 卖 → 关闭 → 恢复
  • 结算画面：玩家死亡或通关 → 暂停 → 显示本局结果 → 重开 / 返回主菜单

口诀：所有"游戏要停、玩家要做选择"的界面 = 同一个节点结构 + 同一个 Process Mode 设置。

📌 老师注记
  • Process Mode 设在哪一层？设 UpgradePanel 自己，不是 CanvasLayer。CanvasLayer 是 UI 容器，Process Mode 是"我是否受暂停影响"——两个不同维度。
  • 不要每个节点都设 Always：只需要菜单根节点 + 它的子节点会自动继承。子节点设 Inherit 即可。
  • 暂停时动画怎么办？如果升级弹窗想"淡入"动画 → 弹窗的子 AnimationPlayer 也得能跑。Container 类节点会继承 Process Mode，所以子节点不用单独设。
  • 暂停时 Timer 怎么办？Timer 默认是 Pausable，暂停时停。如果升级弹窗自己有倒计时（比如 10 秒不选就默认）→ Timer 也设 Always。
  • 不要一开始就塞太多功能：先做"杀 5 只升级、左右各一按钮"最小版。随机、三选、池子、Resource、Autoload 全部不引入，下一阶段再说。
⚠️ 常见坑点 Common Pitfalls
  • 弹窗显示出来但按钮点不动：Process Mode 没设 ALWAYS（或只设了 CanvasLayer，没设 UpgradePanel）。
  • 暂停时敌人 / 子弹还在动：没调 get_tree().paused = true，只把弹窗 hide 了。检查：触发升级的代码里有没有这行。
  • 恢复后玩家"被推着走"：暂停期间玩家没动，恢复后 move_and_slide() 在暂停时不跑。但 Vector2 累积可能跳一帧——通常无害，知道就好。
  • CanvasLayer 里 process_mode 设了 Always 但弹窗仍停：CanvasLayer 是 Node，不是 Control。它的 Process Mode 不控制子 Control 节点。每个 Control 节点（Button/Label）都从父 Control 继承——但根的 Control 要自己设 Always。
  • 多个弹窗叠在一起：在 Inspector 用 Visible 控制，不要用 show()/hide() 切场景。Hide 的节点不参与输入，但仍占内存。
💡 核心概念
  • get_tree().paused = true：一行代码暂停整个游戏树
  • Process Mode = Always：让 UI 节点在暂停时仍工作
  • 世界 vs 菜单：暂停时该停的 vs 暂停时该继续的 —— 升模块的核心抽象
  • 口诀：世界暂停，菜单继续
🔗 UI 布局放哪学？

本模块管「暂停 + 能点击」，不管「按钮怎么居中、横排」。节点树里的 Container 组合、Full Rect、Button 尺寸 → 见 模块十一：UI 布局。
