模块十一

UI 布局：自己组合，而不是照抄

Container、Button、锚点与 Full Rect —— 学会「积木」，就能搭出任意界面

📋 这节在教什么 Concept
  • 不是 SOP：本模块不会给你一份「升级菜单节点树照着搭」的流水线步骤。那样只会操作，不会设计。
  • 是积木盒：Godot UI 由少数几种积木组成。你学会每块积木「管什么」，就能自己组合出暂停菜单、结算画面、商店、HUD……
  • 先想再搭：打开场景树之前，先回答三个问题——要铺满屏幕吗？要横排还是竖排？要居中吗？
  • 前置知识：CanvasLayer（module-8）——UI 要挂在独立的 GUI 层，不跟 Player 一起走。

一句话：UI 不是背节点树，是选积木、排积木。

🧱 两类控件：普通 Control vs Container

所有 UI 节点都继承自 Control。但要先分清两类——它们的「管孩子方式」完全不同。

普通 Control（自己管自己，不管孩子怎么排）

ColorRect     色块 / 半透明遮罩
Button        可点击按钮
Label         文字
TextureRect   图片

子节点挂在它们下面时，不会自动排版——要靠锚点（anchors）和 offset 手动定位。

Container（会帮子节点排版）

HBoxContainer     把孩子横着排成一行 →
VBoxContainer     把孩子竖着排成一列 ↓
CenterContainer   把一个孩子放到正中央 ⊙
PanelContainer    给孩子加边框 / 背景，再包住内容
MarginContainer   给孩子四周加留白边距

Container 的子节点不要用锚点手动摆——父容器会按自己的规则安排位置和大小。

判断法则：这个节点需要「帮孩子们排队」吗？要 → Container；只是画颜色 / 显示文字 → 普通 Control。

🔘 Button：交互积木
  • 干什么：显示一段文字（或图标），等玩家点击。点击时发出 pressed 信号。
  • Text：按钮上显示什么——「继续」「移速 +20%」「退出」。
  • Custom Minimum Size：按钮至少多大。不设的话，按钮会缩成一个小点，看起来像「布局坏了」。
  • Size Flags：在 Container 里「要不要抢多余空间」。默认不抢；做弹簧占位时才勾 Expand。
  • 信号：在 Node 面板把 pressed 连到脚本，或在代码里 button.pressed.connect(func)。

Button 自己不会居中、不会横排——这些交给外面的 Container 做。

📦 常用 Container 详解

HBoxContainer——横向一排

  • 子节点从左到右排成一行。
  • Separation：两个孩子之间的间距（像素）。
  • Alignment：在这一行的垂直方向怎么对齐（Begin / Center / End）。
  • 适合：左右两个选项、一排技能图标、「取消 | 确认」。

VBoxContainer——纵向一列

  • 子节点从上到下排成一列。
  • Separation：上下间距。
  • Alignment：在这一列的水平方向怎么对齐。
  • 适合：暂停菜单「继续 / 设置 / 退出」、结算画面的标题 + 分数 + 按钮。

CenterContainer——正中放一个

  • 只擅长居中一个孩子（通常再套 HBox 或 VBox 放多个元素）。
  • 自己需要足够大（通常 Full Rect 铺满父节点），才有「大的中央」可以放东西。
  • 适合：弹窗整体居中、标题居中、单个对话框。

PanelContainer——加框 / 底

  • 不改变排列方向，只给孩子加主题边框和背景，让选项看起来像「卡片」。
  • 常套在 Button 外面：PanelContainer → Button。
  • 适合：升级选项卡、物品格子、带边框的按钮组。
📐 Full Rect 与锚点：铺满屏幕

很多 UI 需求的第一步是：这个节点要盖住整个屏幕（或整个父区域）吗？ 要 → Full Rect。

锚点是什么

  • 锚点定义控件的四条边「粘在父节点的哪条比例位置」。
  • Left=0, Top=0, Right=1, Bottom=1 → 四边贴满父节点 = Full Rect。
  • Offset 是锚点定好之后的微调边距（通常全 0）。

Godot 4 里怎么设 Full Rect

  • 2D 视图顶部工具栏 → Layout 按钮（格子图标）→ Full Rect。
  • 或右键节点 → Layout → Full Rect。
  • 或 Inspector → Control → Layout → 手动填 Anchor 0, 0, 1, 1，Offset 全 0。

⚠️ Inspector 里经常看不到「Anchors Preset」按钮——这是 Godot 4 的正常现象，去 2D 视图顶部的 Layout 菜单找。

三种 Layout Mode（Inspector → Layout）

Position   用手动坐标 (offset_left / offset_top …) 定位
Anchors    用锚点 + offset 定位（设 Full Rect 用这个）
Container  父 Container 托管大小——作为 Container 的子节点时自动切换

作为 Container 的子节点时，没有锚点预设——大小由父容器决定。作为 ColorRect 的子节点时，是普通 Control，要用 Anchors 或 2D 视图拖矩形。

🌫️ ColorRect：遮罩，不是排版器
  • 干什么：画一块纯色矩形。调 Color 的 Alpha 透明度 → 半透明暗色遮罩。
  • 常见用法：弹窗后面的「变暗背景」，通常设 Full Rect 铺满屏幕。
  • 它不是 Container：挂在 ColorRect 下的孩子不会自动居中或横排——还得靠 CenterContainer / HBoxContainer 等。
  • 结构建议：遮罩和菜单内容可以是兄弟节点（都挂在 CanvasLayer 下），不必把所有东西塞进 ColorRect 里面。这样少踩「child of regular control」的坑。
🧩 组合思路：需求 → 积木（不是标准答案）

下面不是「唯一正确节点树」，而是设计推理——你可以换顺序、换 Container 类型，只要达到同样的视觉效果。

需求 A：全屏变暗 + 屏幕正中左右两个大按钮

  • 要铺满？→ ColorRect，Full Rect，半透明。
  • 要正中？→ CenterContainer，Full Rect。
  • 要横排两个？→ CenterContainer 里放一个 HBoxContainer，Separation 拉开间距。
  • 要卡片感？→ 每个 Button 外套 PanelContainer。
  • 按钮太小？→ 给 Button 设 Custom Minimum Size。

需求 B：屏幕正中竖着三个按钮（暂停菜单）

  • 遮罩同上。
  • CenterContainer → VBoxContainer（不是 HBox）→ 三个 Button。
  • VBox 的 Separation 控制按钮间距。

需求 C：分数在左上角（HUD）

  • 不需要 CenterContainer。
  • CanvasLayer 下直接放 Label，锚点设左上角（Top Left），offset 留一点边距。
  • HUD 通常不需要 Full Rect——只占一小块。

需求 D：一行内容水平居中，但不用 CenterContainer

  • HBoxContainer 设 Full Rect，Alignment = Center（垂直居中）。
  • 左右各放一个空的 Control，勾 Horizontal Expand → 当弹簧，把中间内容挤到水平正中。

口诀：先拆需求（铺满？居中？横排？竖排？），再选积木，最后想嵌套顺序。

✏️ 动手前：问自己这 5 个问题

搭任何界面前，在纸上或脑子里回答完再打开 Godot——比照着别人的节点树抄快得多。

  1. 根节点放哪？ → CanvasLayer（module-8），跟 Player 同级。
  2. 要不要盖住全屏？ → 要：ColorRect + Full Rect；不要：Label / 小 Panel 锚角落即可。
  3. 主要内容横排还是竖排？ → HBox 或 VBox。
  4. 整体要不要在屏幕正中？ → CenterContainer（或 HBox + 弹簧）。
  5. 每个可点击区域多大？ → Button 的 Custom Minimum Size + 可选 PanelContainer 包一层。
🌲 嵌套规则 Cheat Sheet
CanvasLayer （GUI 根，不随相机动） ├── ColorRect（遮罩） Full Rect · 只管画颜色，不管排版 └── CenterContainer Full Rect · 把下一个孩子居中 └── HBoxContainer 或 VBoxContainer 真正排按钮的地方 ├── PanelContainer → Button └── PanelContainer → Button

↑ 这是一种常见组合，不是标准答案。你也可以把 CenterContainer 和 ColorRect 做成兄弟节点，或省略 PanelContainer，或用 MarginContainer 加边距——能跑、好看、你讲得清为什么这样搭，就是好设计。

⚠️ 布局常见坑点
  • 按钮缩成小点：没设 Custom Minimum Size，也没设 Text。
  • 菜单挤在左上角：遮罩或 CenterContainer 没 Full Rect；或父节点本身只有 40×40 大。
  • Inspector 找不到 Anchors Preset：去 2D 视图顶部 Layout；或父节点是 Container 时子节点本来就没有预设。
  • 「child of a regular control」蓝字提示：节点挂在 ColorRect 等普通 Control 下，要用锚点定位，不能指望父节点帮你排版。
  • CenterContainer 里塞了三个 Button：CenterContainer 只居中一个孩子；多个 Button 要先包进 HBox / VBox，再把 HBox 作为 CenterContainer 的唯一子节点。
  • HBox 设了 Full Rect 但内容还在左边：HBox 默认把孩子靠左排；要水平居中整组 → 加 CenterContainer，或用左右弹簧 Control。
  • UI 跟着玩家跑：CanvasLayer 错挂成 Player 的子节点（module-8 反面教材）。
  • 代码里写 size = Vector2(1280, 720)：换分辨率就歪；优先用锚点 Full Rect，别硬编码像素。
📌 老师注记
  • 别课上念节点树：让学生先画草图（遮罩 / 居中 / 横排），再打开 Godot 选 Container。讲评时问「你为什么用 HBox 而不是 VBox」，而不是「你节点名跟讲义不一样」。
  • 模块 10 + 模块 11 分工：模块 10 管「暂停时 UI 还能点」；模块 11 管「UI 长什么样、怎么排」。可以分开教，也可以交叉引用。
  • 验收标准：学生能口头解释每层 Container 的职责；界面在 1280×720 下居中可读；按钮能点。不要求与讲义节点树一致。
  • 进阶但不必修：MarginContainer、GridContainer、Size Flags 弹簧、Theme 换肤——会了 HBox/VBox/Center 已够做升级菜单和暂停菜单。
  • 实操建议：先搭「空壳 UI」（只有色块和按钮文字），跑通布局和点击，再接 module-10 的 paused 和 buff 逻辑。
💡 练手：用积木自由搭（无标准答案）

下面每题只描述视觉效果，不给出节点树。搭完后对照「练手自检」。

  • 练习 1：半透明遮罩 + 屏幕正中央一个写着「继续游戏」的大按钮。
  • 练习 2：同上遮罩 + 正中央横排两个选项：「移速 ↑」「射速 ↑」，中间有明显间距。
  • 练习 3：无遮罩，左上角显示「Score: 0」，不挡游戏画面。
  • 练习 4：正中央竖排三个按钮：「继续」「设置」「退出」。

练手自检（搭完逐条打勾）：

  • 换窗口大小时，遮罩仍铺满？（用了锚点而不是写死 size）
  • 按钮大小够点、文字看得见？
  • 能说出每一层 Container「负责哪一件事」？
  • 删掉某一层，布局会怎么坏？（说明你真的理解嵌套）

搭得像不像讲义不重要；讲得清「为什么这样排」才重要。
